Why Another Socket Library?

Why not? It would be good if there is a simple and elegant socket library that you can use in projects written in different programming languages.

Mocket's programming style is heavily influenced by JavaScript. Programmers should be able to focus more on business logic instead of network detail when using Mocket.

Mocket implements a client-server connection model. Mocket clients need to know the IP addresses and port number of Mocket servers. Mocket generally assumes each client only connect to one server but one server will maintain connections from multiple clients. However, more flexible designs are easy to achieve programmatically

Mocket is event driven. All Mocket messages are associated with an event type as string. Mocket clients and servers can filter socket packages by event types.
